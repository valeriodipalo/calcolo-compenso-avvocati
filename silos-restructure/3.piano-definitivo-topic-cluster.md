# Piano Definitivo: Topic Cluster — AvvocatoTools.it

**Status**: FINAL
**Date**: 2026-04-05
**Sources**: 1A.claude-approved plan + 2.Manus operational guide + content-plan.csv

---

## Context

AvvocatoTools.it has 10 LIVE legal calculator tools, all at flat root-level URLs with no topical organization. The site needs topic clusters to build SEO topical authority and improve user navigation between related tools. This plan unifies three strategy documents into an actionable implementation roadmap, keeping existing URLs intact and adding cluster infrastructure on top.

---

## Confirmed Decisions

| Decision | Choice | Rationale |
|---|---|---|
| **URLs** | Stay flat, no 301 redirects | Preserve existing rankings, clusters via linking not URL hierarchy |
| **Cluster taxonomy** | 8 clusters (Manus definitions) | Complete vision, only render clusters with LIVE tools |
| **Navbar** | Dropdown per macro-area | Multiple dropdowns, each contains cluster groups with tools inside |
| **Homepage** | "Capitoli" with visible tool cards | Cluster sections with H2, description, tool cards below |
| **CU Hub URL** | `/guida-contributo-unificato` | Avoids collision with `/contributo-unificato` tool |
| **Pillar pages** | Content-rich editorial with internal linking | 2000+ words, tool grid, editorial sections, FAQ |
| **Sequencing** | Registry + components first, DFS when creating pillar pages | Steps 1-6 don't depend on keyword data |

## Part 2: Navbar — Dropdown per Macro-Area

Pattern: multiple dropdown buttons, each containing cluster groups with tools listed inside.

**Current concept** (refine during implementation):

```
Desktop: [Processo ▾] [Calcoli ▾]      (future: +[Fiscale ▾] +[Utility ▾])

"Processo" dropdown:
├── COMPENSI
│   ├── Compenso Avvocato
│   └── Fattura Avvocato
├── TERMINI
│   ├── Termini Processuali
│   └── Termini Impugnazione
└── CONTRIBUTO UNIFICATO
    └── Calcolo CU

"Calcoli" dropdown:
├── INTERESSI E RIVALUTAZIONE
│   ├── Interessi Legali
│   ├── Interessi Moratori
│   └── Rivalutazione Monetaria
└── DANNO E RISARCIMENTO
    ├── Danno Biologico
    └── Danno Non Patrimoniale
```

**Mobile**: Accordion with macro-areas as top-level, clusters as sub-sections.

**Implementation notes:**
- Hover-triggered dropdown (desktop) with 150ms close delay
- Active macro-area highlighted when current page is in its scope
- Cluster labels become links to hub pages when `hubReady: true`
- New macro-areas auto-appear when clusters 6-8 get LIVE tools
- Exact macro-area grouping to be refined during implementation

**Data model addition**: `NavGroup` interface linking macro-areas to clusters:
```typescript
interface NavGroup {
  id: string;
  label: string;        // "Processo", "Calcoli"
  clusterIds: string[];  // ["compensi", "termini", "contributo"]
}
```

## Part 4: Breadcrumb Strategy

**Visual** (between navbar and hero):
```
Home  ›  Interessi e Rivalutazione  ›  Calcolo Interessi Legali
```

- Level 1 ("Home") → `/`
- Level 2 (cluster label) → hub page when `hubReady`, else `/#cluster-id`
- Level 3 (tool name) → current page (not a link)

**JSON-LD BreadcrumbList** (3 levels):
- Level 2 URL always points to hub slug (even before hub exists)
- Uses existing `breadcrumbSchema()` function in `src/lib/schema.tsx`

**Component**: `src/components/shared/Breadcrumb.tsx`
- Server component, props: `currentSlug: string`
- Renders both visual HTML and JSON-LD
- Integrated into ToolLayout (automatic for all tools)

## Part 5: Internal Linking

### Same-cluster (automatic via RelatedTools component)
Every tool page shows siblings from same cluster.

### Cross-cluster (manual via relatedSlugs)
| Tool | Cross-cluster | Rationale |
|---|---|---|
| calcolo-compenso | calcolo-fattura-avvocato | Fee → invoice |
| calcolo-fattura-avvocato | calcolo-compenso | Reverse |
| calcolo-danno-biologico | calcolo-interessi-legali, calcolo-rivalutazione-monetaria | Damages → revaluation + interest |
| calcolo-danno-non-patrimoniale | calcolo-interessi-legali, calcolo-danno-biologico | Same damage workflow |
| calcolo-interessi-legali | calcolo-rivalutazione-monetaria | Often calculated together |
| calcolo-interessi-moratori | calcolo-interessi-legali | Users confuse the two types |
| calcolo-rivalutazione-monetaria | calcolo-interessi-legali, calcolo-danno-biologico | Used in damages |
| calcolo-termini-processuali | calcolo-termini-impugnazione | Deadline → appeal |
| calcolo-termini-impugnazione | calcolo-termini-processuali | Reverse |
| contributo-unificato | calcolo-compenso | CU + fee for same case |

### Hub linking (when pillar pages exist)
- Hub → tools: editorial contextual links + tool card grid
- Tools → hub: breadcrumb level 2 + optional in-text CTA

### 3-link minimum per tool (Manus rule)
1. Breadcrumb → hub (1 link)
2. RelatedTools → same-cluster (1-2 links)
3. RelatedTools → cross-cluster (1-2 links)

## Part 6: Pillar/Hub Page Structure

Each pillar page is an editorial guide (2000-3000 words):

```
[Breadcrumb: Home › Cluster Label]
[H1: keyword-rich informational title]
[Intro: 200-300 words — overview, audience, normative framework]
[Tool Grid: cards linking to each tool in cluster]
[H2: Topic 1 — 500-700 words, contextual link to tool]
[H2: Topic 2 — 500-700 words, contextual link to tool]
[H2: Topic 3 — 500-700 words, contextual link to tool]
[H2: Comparison/disambiguation table]
[H2: Practical scenarios — cross-links to other clusters]
[FAQ: 10-15 questions from PAA + long-tail keywords]
[RelatedTools: cross-cluster suggestions]
[Fonti e Riferimenti]
```

**Schema**: BreadcrumbList (2 levels) + Article. NO WebApplication, NO FAQPage.

**Priority order**:
1. `interessi-e-rivalutazione` — 3 tools, dense cluster
2. `compensi-forensi` — highest CPC, core audience
3. `termini-processuali` — highest SV (60K)
4. `danno-e-risarcimento` — 2 tools, clear intent
5. `guida-contributo-unificato` — massive impressions

**When a hub goes live**:
1. Set `hubReady: true` in CLUSTERS
2. Breadcrumb level-2 auto-updates to link to hub
3. Navbar cluster label becomes clickable
4. Homepage section gets "Scopri di più →"
5. Sitemap auto-includes hub URL
